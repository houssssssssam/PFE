# COOPIYO — API Backend (Laravel 11)

## Project Overview
Coopiyo is a SaaS multiplatform (Web + Mobile) platform connecting users with qualified experts,
assisted by AI. Two-tier model:
- Level 1 (AI): GPT-4 analyzes the request, classifies the domain, answers immediately
- Level 2 (Human Expert): if AI confidence < 0.75 or user requests it, escalate to best available expert

Target: Francophone Africa (Morocco first). Languages: French + Arabic.
Team: 2 full-stack developers. Duration: 27 weeks (27 sprints of 1 week each).

## Tech Stack
- **Framework**: Laravel 11 (PHP 8.3)
- **Database**: PostgreSQL 16
- **Cache / Queue**: Redis 7
- **Auth**: Laravel Sanctum + JWT refresh tokens (15min access, 30 days refresh httpOnly cookie)
- **OAuth**: Laravel Socialite (Google + Facebook)
- **2FA**: TOTP via Google Authenticator (optional for users)
- **Real-time**: Laravel Reverb (WebSocket) + Laravel Echo client
- **AI Orchestration**: n8n (self-hosted via Docker)
- **LLM**: OpenAI GPT-4
- **Speech-to-Text**: OpenAI Whisper API
- **Text-to-Speech**: ElevenLabs API
- **Vector DB**: Qdrant (self-hosted) for RAG knowledge base
- **Storage**: AWS S3 (or MinIO locally) for audio + documents
- **Payment**: Stripe (international) + CMI (Morocco local)
- **PDF**: DomPDF (invoices)
- **Email**: Brevo HTTP API (NOT Gmail — blocked on Render free tier)
- **Push Notifications**: Firebase Cloud Messaging (FCM)
- **Containers**: Docker + Docker Compose
- **CI/CD**: GitHub Actions
- **Monitoring**: Laravel Telescope (dev), Sentry (prod)

## Architecture
- All API routes prefixed `/api/v1`
- Always use Form Requests for all validation
- Always use API Resources / ResourceCollections for all responses
- Services layer for all business logic (thin controllers)
- Repository pattern for DB queries
- All heavy tasks via Redis queues (AI calls, transcription, emails, TTS, PDF)
- Broadcast all real-time events via Laravel Reverb
- n8n handles all AI workflow orchestration via webhooks
- Never hardcode secrets — always `.env`
- Spatie Laravel Permission for roles (user, expert, admin)
- Policies for authorization on every model

## n8n Workflows (Full Architecture)

n8n is self-hosted via Docker. Laravel communicates with n8n via webhooks.
All webhooks protected with shared secret header: `X-N8N-Secret`.

### Workflow 1 — Message Analysis & Classification
**Trigger**: POST from Laravel when user sends a message
**Steps**:
1. Receive payload (conversation_id, message_id, content, type, language, context)
2. If type = audio → call Workflow 4 (transcription) first
3. Detect language (fr/ar) via GPT-4
4. Classify domain (médecine, droit, fiscalité, etc.) via GPT-4
5. Search Qdrant vector DB for relevant knowledge (RAG)
6. Evaluate complexity → route to Workflow 2 or 3

### Workflow 2 — AI Response Generation
**Trigger**: called from Workflow 1 when AI can answer
**Steps**:
1. Build prompt with: system prompt + RAG context + conversation history
2. Call GPT-4 with confidence scoring
3. If confidence >= 0.75 → generate full response
4. Optionally call Workflow 5 (TTS) for audio response
5. Callback Laravel POST /api/v1/ai/callback with response

### Workflow 3 — Expert Routing
**Trigger**: called from Workflow 1 when confidence < 0.75
**Steps**:
1. Find best available expert: same category + is_available=true + highest rating
2. Assign expert to conversation
3. Callback Laravel to update conversation.status = 'expert'
4. Trigger push notification to expert

### Workflow 4 — Audio Transcription
**Trigger**: called when audio message uploaded to S3
**Steps**:
1. Download audio from S3 presigned URL
2. Send to Whisper API
3. Return transcription text
4. Callback Laravel POST /api/v1/ai/transcription-complete
5. Continue with Workflow 1 using transcribed text

### Workflow 5 — Text-to-Speech
**Trigger**: called from Workflow 2 after AI response generated
**Steps**:
1. Send response text to ElevenLabs API
2. Upload generated audio to S3
3. Callback Laravel POST /api/v1/ai/tts-complete with audio_url

### Workflow 6 — Content Moderation
**Trigger**: every new message
**Steps**:
1. Check content for inappropriate/harmful content via GPT-4
2. If flagged → mark message + notify admin

### Workflow 7 — Conversation Summary
**Trigger**: conversation closed
**Steps**:
1. Fetch all messages from conversation
2. Generate summary via GPT-4
3. Update conversation.summary in DB

## Laravel → n8n Communication

```php
// app/Services/N8nService.php
class N8nService {
    public function triggerWorkflow(string $webhook, array $payload): void
    {
        Http::withHeaders(['X-N8N-Secret' => config('services.n8n.secret')])
            ->timeout(30)
            ->post(config('services.n8n.base_url') . $webhook, $payload);
    }
}

// Webhook URLs (set in .env):
// N8N_WEBHOOK_ANALYZE=/webhook/analyze-message
// N8N_WEBHOOK_MODERATE=/webhook/moderate-content
// N8N_WEBHOOK_SUMMARIZE=/webhook/summarize-conversation
```

## Audio Pipeline

```
User records audio (WebM/Opus browser, M4A/AAC Flutter)
     ↓
Laravel generates S3 presigned URL
     ↓
Client uploads directly to S3
     ↓
Client notifies Laravel: audio_uploaded (message created with media_url)
     ↓
Laravel dispatches TranscribeAudioJob → triggers n8n Workflow 4
     ↓
n8n calls Whisper API → gets transcription
     ↓
n8n callbacks Laravel /api/v1/ai/transcription-complete
     ↓
Laravel updates message.transcription
     ↓
n8n continues with Workflow 1 (AI analysis with transcribed text)
     ↓
n8n optionally calls ElevenLabs TTS for audio response
     ↓
Audio response stored on S3, URL sent back to Laravel
     ↓
Laravel broadcasts AIResponseReady via Reverb
```

## Security

### Authentication
- Sanctum access tokens: 15 minutes expiry
- Refresh tokens: 30 days, httpOnly cookie
- Token rotation on every refresh
- OTP: 6 digits, 10 minutes expiry, email only (no SMS)
- 2FA: TOTP via Google Authenticator (optional)
- OAuth: Google + Facebook via Laravel Socialite

### API Protection
- Rate limiting:
  - Auth endpoints: 10 req/min
  - Regular users: 60 req/min
  - Experts: 200 req/min
  - n8n callbacks: whitelist by IP
- CORS: whitelist frontend URLs only
- All inputs validated via Form Requests
- File uploads: validate MIME type + size (audio: 10MB, docs: 5MB, images: 2MB)
- Sanctum CSRF protection on web routes
- Eloquent only — no raw SQL queries
- XSS: Blade auto-escaping + Content-Security-Policy headers

### Expert Validation (3 steps)
1. Identity verification (CNIE or passport scan)
2. Diploma and certification validation
3. Admin review and approval

### n8n Security
- Shared secret header on all webhook calls
- HTTPS only for all n8n webhooks
- n8n not exposed publicly — only accessible internally or via VPN
