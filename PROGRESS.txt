[S08] n8n + AI WORKFLOWS (branch: feature/ai-workflows)
--------------------------------------------------------
Services:
  - N8nService — HTTP client to n8n webhooks (analyze, moderate, summarize)
    Retry 2x, 30s timeout, safe French fallback responses when n8n is down
  - AiService — orchestrates: moderate -> analyze -> respond -> log -> escalate
    Also: summarizeConversation (called on close)

Jobs:
  - ProcessMessageJob — queued, 3 tries, 5s backoff
    Dispatched from MessageService on every user text message
  - SummarizeConversationJob — queued, 2 tries, 10s backoff
    Dispatched from ConversationCloseController on close

AI Pipeline:
  1. User sends message -> ProcessMessageJob dispatched
  2. Moderation check (flag harmful content)
  3. Analyze: classify intent + generate GPT-4 response
  4. Create AI response message + broadcast AIResponseReady
  5. Auto-escalate if confidence < threshold
  6. Log everything to ai_logs table

Admin endpoints:
  - GET /api/v1/admin/ai-logs (filter by conversation, workflow, escalated)
  - GET /api/v1/admin/ai-logs/stats (totals, tokens, by-workflow breakdown)

Resource: AiLogResource
Routes: 48 (was 46) | All 25 tests pass.
